# CML 파일 포맷 스펙 (파일명 매핑 내역)

## 개요

- **파일 확장자**: `.cml`
- **파일명 예시**: `파일명 매핑 내역.cml`
- **인코딩**: **CP949** (EUC-KR 확장)
- **형식**: `^` 구분자 텍스트, 헤더 1줄 + 레코드 다수
- **총 라인 수**: 약 12,652 (예시 파일 기준)
- **역할**: 클립 ID(N-ID, CM-ID 등) → 아이템명 매핑

---

## 파일 구조

```
#DTV System Cliplist File          ← 헤더 (1줄, 고정)
<type>^<full_name>^<advertiser>^<duration>^<clip_id>^   ← 레코드
...
```

---

## 레코드 기본 형식

```
<type_code>^<full_name>^<advertiser>^<duration>^<clip_id>^
```

| 위치 | 필드명 | 예시 | 설명 |
|------|--------|------|------|
| 0 | `type_code` | `07` | 클립 유형 코드 |
| 1 | `full_name` | `MBM 20초 - MBM 체인저모십(간판화면수정)` | 전체 아이템명 |
| 2 | `advertiser` | `MBM` | 광고주/제작사명 |
| 3 | `duration` | `000020` | 길이 (초, 6자리 0패딩) |
| 4 | `clip_id` | `CM0000003621` | 클립 ID |
| 5 | (빈 필드) | `` | 종료 구분자용 |

## 타입별 레코드 예시

### 타입 07 — CM (상업광고)
```
07^MBM 20초 - MBM 체인저모십(간판화면수정)^MBM^000020^CM0000003621^
07^대방건설 주식회사 30초 - 대방건설광주수완 TV30초 1차^대방건설 주식회사^000030^CM0000004463^
```

### 타입 01 — SB/프로그램 안내 (N-ID)
```
01^24헬로키즈체육교실(재)^^000010^N201406^
01^26건강하우스깡있는아침^^000010^N270897^
```
- `advertiser` 필드가 빈 경우 많음

### 타입 02 — ID/지역 홍보 (N-ID)
```
02^ID여수세계섬박람회007-섬진강^^000015^N275037^
```

### 타입 07 (SB 교통/공공)
```
07^교통안전(전남)30초(5월11일~30일)^^000030^N281021^
```
- 타입 07이지만 N-ID를 사용하는 경우도 존재 (376개)

### 타입 04 — 공익광고
```
04^L)재난 -1폭염-1분48초^2017^000148^CT_V000000037034^
```
- full_name 앞에 `L)` 접두어

### 타입 03/05/06 — 방송 순서 밑그림
```
03^F)방송순서밑그림(秋: 11월用) - 10'4'^(秋: 11월用) - 10'4'^001004^F0000065698^
```
- full_name 앞에 `F)` 접두어
- duration이 초 단위로 100초 이상 가능

### 타입 -1 — CM 그룹 헤더
```
-1^MBC뉴스투데이 2부-대박세,빛장례^CM Group^000040^01월-광고03^
```
- 특정 프로그램 시간대의 CM 그룹 정의
- `advertiser` = `CM Group` (고정)
- `clip_id` = 그룹명 (예: `01월-광고03`)

### 타입 -7 — CM 그룹 상세
```
-7^01월-광고03^CM0000204281^대박세일(26.04)^^00000000^00002000^0^
```
- 확장 필드 구조 (6개 이상 필드)
- 필드[1]: CM 그룹명
- 필드[2]: CM-ID
- 필드[3]: CM 아이템명

| `N######` | `N270944`, `N281021` | SB/프로그램 안내, DDR1 클립 |

| `CT_V###########` | `CT_V000000037034` | 공익광고 |

| `K######` | `K0000001234` | 기타 |

## N-ID 매핑 (DDR1 연결)

DDR1 로그의 `Play CLIP : N######` 이벤트와 CML 파일 연결:

```
DDR1 Log: Play CLIP : N270944
    ↓
CML 조회: clip_id = 'N270944'
    ↓
full_name: "26건강하우스깡있는아침" (예시)
advertiser: ""
duration: 000010
```

### 매핑 쿼리 예시
```python
# CML 파싱 후 딕셔너리 생성
clip_map = {}
for line in cml_lines[1:]:
    parts = line.strip().split("^")
    if len(parts) >= 5:
        clip_id = parts[4]
        full_name = parts[1]
        advertiser = parts[2]
        duration = int(parts[3]) if parts[3].isdigit() else 0
        clip_map[clip_id] = {
            "type": parts[0],
            "full_name": full_name,
            "advertiser": advertiser,
            "duration_sec": duration
        }
```

## 파싱 시 주의사항

1. **인코딩**: 반드시 `cp949`로 읽어야 함 (`euc-kr` 불가)
2. **헤더 라인**: 첫 줄 `#DTV System Cliplist File`은 스킵
3. **빈 라인**: 건너뜀
4. **타입 -7**: 필드 수가 9개로 다른 타입과 다름
5. **N-ID 중복**: 동일 N-ID가 여러 타입에 걸쳐 나타날 수 있으므로 type_code 우선순위 고려
6. **duration 파싱 오류**: 일부 레코드의 duration 필드에 텍스트가 포함된 경우 있음 (파싱 예외 처리 필요)
