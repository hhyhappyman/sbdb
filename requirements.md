# Requirements — SB 송출 대시보드

## 프로젝트 개요

방송국의 SB(Station Break) 송출 내역을 수집·정리하고, 대시보드와 리포트 기능을 제공하는 웹 애플리케이션.

---

|------|------|
| 백엔드 | Python + FastAPI |
| 프론트엔드 | React (Vite + Ant Design + Recharts) |
| DB | SQLite (apst.db / ddr1.db) |
| PDF 생성 | reportlab |

## 데이터 소스

### 소스 1 — APST 파일 (자동 송출)
- 형식: JSON (`.apst`)
- 저장 주기: 매일 1개 이상 파일 (예: `20260506AAA.apst`, `20260506PPP.apst`)
- SB 아이템 식별: `SrcID`가 `N######` 형식 + `Con = K` 또는 `Con = I` ✅ 수정 (과거 `Con = IDC` → `Con = I`)
- SB 띠 구조: `lstGrp` 안에서 두 그룹이 짝을 이룸
  - 그룹1(`EvtGrp.eTy="S"`): 이어서(`C`) → 광고그룹(`G`, 있을 때만) → 캠페인(`K`)* → ID(`I`) 소재 목록
  - 그룹2(`EvtGrp.eTy="P"`, 그룹1과 동일 `eID`): 프로그램(`AIR`) 소재 1개 — 이 그룹의 `EvtGrp.eName`이 SB 띠의 정식 그룹명이며 `program_block`으로 저장
- 상세 포맷: `spec_01_apst_format.md` 참조

### 소스 2 — DDR1 로그 (수동 송출)
- 형식: 텍스트 로그 (`.Log`)
- 저장 주기: 매일 1개 파일
- SB 이벤트 추출: `Play CLIP : N######` 라인
- 상세 포맷: `spec_02_ddr1_log_format.md` 참조

### ✅ 수동 송출 구간 추출 (신규)
- APST의 이어서(`Con=C`) 소재의 주장비명(`MM`)이 `DDR1`이면 해당 SB 띠는 수동으로 송출된 것
- 이 경우 APST에는 캠페인/ID 소재가 비어있거나 실제와 다를 수 있어, DDR1 로그를 직접 분석해 실제 송출 내역을 추출:
  1. 띠 시작 시간(이어서 소재 OnT 시간) 기준 **±5초** 오차 허용 범위 내에서 `Play CLIP : <이어서 clip_id>` 라인 탐색 (시간 동기화 오차 보정)
  2. 그 지점부터, 짝이 되는 프로그램(AIR) 그룹 시작 시간까지 `OnAir ID : <clip_id>` 전환 이벤트를 순서대로 수집 — 각 전환 시점이 해당 소재의 실제 송출 시작 시간
  3. 마지막 소재는 다음 `OnAir ID` 전환이 없을 수 있음(중간에 정지 등) — 이미 OnAir로 전환된 마지막 소재까지만 인정
  4. 각 clip_id는 CML 매핑으로 소재명 변환 — **CML에 없는 clip_id는 소재명 대신 파일명(clip_id)을 그대로 기록** (건너뛰지 않음)
- 추출된 소재는 ddr1.db에 저장되며, `source_file`은 `manual:<로그파일명>:<트리거clip_id>` 형식으로 구분 (일반 DDR1 적재와 중복 검사 범위 분리)

### ⚠️ 레거시 DDR1 전체 로그 스캔 폐기 (2026-06)
- 과거에는 DDR1 로그 전체에서 독립적인 `Play CLIP : N######` 라인을 찾아 캠페인/ID로 분류해 저장했으나, 실데이터 검증 결과 **신뢰할 수 없어 폐기**:
  - 운영자가 단발성으로 미리보기/테스트 재생한 클립도 `Play CLIP` 라인을 남겨 오탐 발생 (다음 소재로 이어지지 않음에도 진짜 송출로 잘못 기록)
  - 단발성 여부를 연속성으로 검증해도, DDR1 장비가 정상 자동 송출 SB 띠까지 기록하고 있어 apst.db와 중복되는 데이터가 생김
- 이제 수동 송출 내역은 **오직 APST `MM=DDR1` 앵커 기반 구간 추출**(위 항목)을 통해서만 적재됨. `Play CLIP` 단독 매칭 방식(`parse_ddr1()`)은 항상 빈 결과를 반환하도록 변경 (하위 호환을 위해 API는 유지)

### 소스 3 — CML 매핑 파일 (클립 ID → 이름)
- 형식: `^` 구분자 텍스트 (`.cml`), 인코딩 **CP949**
- 역할: N-ID / CM-ID → 아이템명 변환
- 상세 포맷: `spec_03_cml_format.md` 참조

## 기능 요구사항

### F-01. 대시보드 (메인 화면)
- 좌측: SB 소재별 총 송출 횟수 테이블
  - 컬럼: 순위 / 소재명 / 종류 / 횟수 / **SA / A / B / C** (급지별 횟수)
  - 페이지당 표시 개수 선택: 10 / 20 / 30 / 40 / 50개
  - 페이지 이동 시 순위 번호 연속 유지 (예: 2페이지 → 11번부터)
- 우측: 시간대별(0~23시) 송출 횟수 막대 차트
  - **차트 높이는 좌측 페이지 크기 선택과 무관하게 항상 고정** ✅
- 필터: 년도 + 월 + 소재종류(전체/캠페인/ID)
- **탭 이동 후 복귀 시 마지막 검색 결과 유지**

### F-02. 달력 뷰 ✅ 대개편 완료
- 월별 달력 표시, 날짜별 송출 건수를 셀 중앙에 배지로 표시
- 달력 헤더: 년도 + 월 + **소재종류(전체/캠페인/ID, 기본값 캠페인)** 선택
- **날짜 숫자 표시**: 앞자리 0 없이 한 자리로 표시(예: "01"이 아닌 "1"), 폰트 크기 기존 대비 30% 축소
- **토요일·일요일·법정공휴일**: 날짜 숫자 빨간색 표시 (음력 환산 공휴일 포함 근사 데이터 내장)
- **날짜 클릭 시 선택 표시**: 클릭한 날짜 셀에 시안색(cyan) 배경+테두리로 강조. antd 기본 "선택됨" 표시는 무력화하고 직접 관리하는 `selectedDate` 상태로 제어
- **상세 패널 — Drawer 미사용, 리사이즈 가능한 분할 레이아웃으로 변경** ✅
  - 달력(좌측)과 상세 패널(우측)이 같은 행에 나란히 배치 — 패널이 열려도 달력이 가려지지 않음
  - 패널 너비는 경계선을 마우스로 드래그하여 조절 가능 (달력 최소 420px, 패널 최소 360px 보장)
  - 패널 닫기는 카드 우상단 X 버튼
- 상세 패널 내용 (날짜 클릭 시, 위→아래 표시 순서) ✅ 순서 변경:
  1. **소재별 송출 횟수**(집계 요약, 송출내역을 클라이언트에서 그룹화): 소재명 / 종류 / 총횟수 / SA / A / B / C — 소재명·종류·총횟수 정렬 토글, 기본 정렬은 총횟수 내림차순
  2. **송출내역**(시간 순서 목록): 시간 / 소재명 / 종류 / 구분 — 모든 컬럼 클릭 시 오름차순/내림차순 정렬 토글, 기본 정렬은 시간 오름차순
  3. 두 결과 화면 각각 제목 옆에 **CSV 저장 버튼** 제공 (UTF-8 BOM 포함, 엑셀 한글 깨짐 방지)
- 소재종류 필터 변경 시 이미 열려있는 상세 패널도 새 필터로 재조회
- **탭 이동 후 복귀 시**: 선택했던 날짜·패널 결과·선택 강조가 모두 그대로 유지

### F-03. 상세조회
- 기간: 시작일 ~ 종료일 (연·월·일 모두 선택, 달력 DatePicker 포함)
- 송출시간: 시작 시 ~ 끝 시 (지정 안 하면 24시간 전체)
- 소재종류: 전체 / 캠페인 / ID
- 송출구분: 전체 / 자동(APST) / 수동(DDR1)
- 소재명 검색 ✅ 정확매칭 방식으로 확정
  - 입력 없으면 전체 조회
  - 조회 시 먼저 `/api/items`로 후보를 찾고, **정확히 1개만 매칭되면 바로 그 소재로 조회**
  - 0개 또는 2개 이상 매칭되면 **선택 모달**을 띄워 후보 목록(소재명/종류/총횟수)을 보여주고 클릭으로 선택
  - 최종 조회는 항상 선택된 소재명과 **완전일치**로 필터링 (부분일치 X — 예: "무등산" 선택 시 "무등산캠페인" 등 다른 소재는 제외)
- **탭 이동 후 복귀 시 마지막 검색 결과 유지**

### F-04. 소재별 월 리포트 + PDF 저장 ✅ 레이아웃 확정
- 소재명 입력 방식:
  1. 직접 타이핑 가능
  2. 조회 시 **정확히 일치하는 소재가 있으면 바로 결과 조회**
  3. 정확히 일치하는 소재가 없으면 **포함 소재 목록 모달** 표시 → 선택
- 년도 + 월 선택 후 날짜별 송출 시간 테이블 표시
- PDF 저장: `reports/monthly/` 폴더, `sb_monthly_{소재명}_{YYYYMM}.pdf`
- **탭 이동 후 복귀 시 마지막 검색 결과 유지**

#### F-04 PDF 레이아웃 ("광주MBC 방송홍보 SB송출 현황" 양식)

**① 헤더**: 우상단 광주MBC 로고 + 대형 제목

**② 정보 테이블 (4행)**
| 레이블 | 값 | 레이블 | 값 |
|--------|----|--------|----|
| 회 사 명 | 광고주명 | 사업자등록번호 | 등록번호 |
| 송출 내용 | SB 아이템명 | 대 표 이 사 | 대표이사명 |
| 송출 매체 | TV, 1일 N회 | 업 태·업 종 | 업태·업종 |
| 송출 기간 | YYYY.MM.DD(요일)~ | 비 고 | 송출시간은 방송사 상황에 따라 변동될 수 있음 |

→ 광고주 정보: **광고주 관리 메뉴**에서 입력
→ 로고·직인 이미지: **환경설정 메뉴**에서 파일 경로 지정

**③ 본문 데이터 테이블**
- 컬럼: `일시` | `요일` | `횟수` | `TV` | `RADIO-AM` | `RADIO-FM` | `비고`
- TV 시간: `HHMM` 형식, 자정 이후 **25시제** 적용 (예: 01:14 → `2514`)
- RADIO: 현재 `-` 표시 (미사용)
- 마지막 행: 총 계

**④ 푸터**: 확인 문구 + 광주문화방송(주) + 직인 이미지

### F-05. 텔레그램 챗봇 (Phase 5 예정)
- `/report 소재명 2026 05` → PDF 파일 전송
- `/count 소재명 2026 05` → 텍스트 응답
- `/help` → 사용 방법 안내

### F-06. 방송 운행표 (구 "일별 SB내역", 송출내역 출력 메뉴) ✅ 명칭 변경
- 날짜 선택 → 해당 날짜의 APST 파일을 `apst_dir`에서 자동 검색
- 결과를 **화면에 먼저 표시** (테이블: 방송시작시간 / 프로그램명 / SB 소재 제목)
- 별도 **PDF 저장** 버튼 제공
- PDF 저장 위치: `reports/daily/sb_report_YYYYMMDD.pdf`
- **탭 이동 후 복귀 시 마지막 결과 유지**

### F-07. 일별 재난방송 소재 (송출내역 출력 메뉴)
- 날짜 선택 → `_PgmName_`에 **"재난"** 포함 소재만 필터
- 결과를 **화면에 먼저 표시**
- 별도 **PDF 저장** 버튼 제공
- 재난 소재가 없을 경우: **"해당 없음" 문구가 적힌 빈 PDF 생성** ✅
- PDF 저장 위치: `reports/disaster/disaster_report_YYYYMMDD.pdf`
- **탭 이동 후 복귀 시 마지막 결과 유지**

### F-08. 일일 운행표 (캠페인 전용, 송출내역 출력 메뉴) ✅ 신규
- 날짜 선택 → 해당 날짜의 **캠페인 소재만** 집계
- 컬럼: 소재명 / 총횟수 / SA / A / B / C
- 소재명·총횟수 클릭 시 정렬 토글 (기본: 총횟수 내림차순)
- 별도 **PDF 저장** 버튼 제공 (총계 행 포함)
- PDF 저장 위치: `reports/summary/campaign_summary_YYYYMMDD.pdf`

### F-09. 일일 ID 운행표 (ID 전용, 송출내역 출력 메뉴) ✅ 신규
- F-08과 동일한 구조, **ID 소재만** 집계
- PDF 저장 위치: `reports/summary/id_summary_YYYYMMDD.pdf`

- 관리자 로그인 후: 관리자 전용 메뉴 3개 추가 표시
- 직접 URL 접근 시에도 관리자 전용 페이지는 "권한 없음" 화면 표시

## 파일 적재 방식 ✅ 구현 완료

### 방식 1 — 실시간 폴더 감시 (watchdog)
- 환경설정에서 지정한 `apst_dir`, `ddr1_dir` 폴더를 실시간 감시
- 새 파일 도착 시 자동 파싱·DB 적재
  - `.apst` → APST DB 적재
  - `.Log` → DDR1 DB 적재 (파일명에서 날짜 자동 추출)
  - `.cml` → clip_map 자동 갱신
- 감시 ON/OFF: 웹 UI 스위치로 제어
- 서버 재시작 시 이전 감시 상태 자동 복원 (DB에 상태 저장)
- 이벤트 로그: 최근 100건 메모리 보관, UI에서 조회 가능

### 방식 2 — 수동 스캔

### 방식 3 — 단일 파일 업로드
- 파일 선택 → 업로드 → 즉시 적재

## 급지(SA/A/B/C) 분류 ✅ 신규 확정

송출시간에 따라 모든 캠페인·ID 소재를 4단계 급지로 자동 분류하여 DB에 저장.
**시작시간은 이상(≥), 끝시간은 미만(<) 기준**으로 구간을 판정.

| 급지 | 시간 구간 |
|------|----------|
| SA | 19:30 ~ 23:30 |
| A  | 08:30~09:30, 19:00~19:30, 23:30~24:00 |
| B  | 07:00~08:30, 09:30~12:00, 18:00~19:00, 00:00~01:00 |
| C  | 01:00~07:00, 12:00~18:00 |

- ✅ 수정: 12:00~18:00 구간을 **C급**으로 포함 (과거에는 미분류 NULL이었으나, 총횟수와 SA+A+B+C 합계가 불일치하는 문제로 수정됨) — 이제 모든 24시간이 SA/A/B/C 중 하나로 분류되어 미분류 없음
- APST·DDR1 양쪽 파서에서 `broadcast_time` 기준으로 계산 후 `grade` 컬럼에 저장
- 기존 적재 데이터는 서버 시작 시 자동 backfill (마이그레이션)
- 대시보드·달력·일일 운행표·일일 ID 운행표에서 급지별 횟수로 시각화

## 소재종류 정의 ✅ 확정 (수정)

| 소재종류 | APST Con 값 | DB 저장 | 보고서 대상 |
|---------|------------|--------|-----------|
| 캠페인 | `K` | ✅ | ✅ |
| ID | `I` ✅ 수정(과거 `IDC`) | ✅ | ✅ |
| 이어서 | `C` ✅ 수정(과거 `I`) | ❌ | ❌ |
| 광고그룹 | `G` | ❌ | ❌ |
| 프로그램 | `AIR` ✅ 수정(과거 `PGM`으로 오인) | ❌ | ❌ |

## 집계 규칙 ✅ 확정 (수정)

### APST
- `Con=K` → 캠페인
- `Con=I` → ID ✅ 수정 (과거 `Con=IDC`였으나 실제 데이터에는 `IDC`가 거의 없어 ID 소재가 누락되던 버그 — `Con=I`로 정정)
- `Con=C`(이어서) / `Con=G`(광고그룹) / `Con=AIR`(프로그램) → 저장 안 함

### DDR1
1. CML 소재명에 "ID" 포함 → **ID**
2. APST DB 캠페인 이름 목록과 일치 → **캠페인**
3. 그 외 → 저장 안 함

## 2026-07 요구사항 추가·변경

### R-1. 공익/재난 소재 분류 (재정비)
- APST 적재 시 소재종류가 **PRM·R·F·E 등이면서 소재명에 '공익'/'재난'** 포함 → `공익재난`으로 저장
- **K(캠페인)** + 공익/재난 → 기본은 캠페인 유지, 단 프로그램이 **'방송 종료 안내'**면 공익재난으로 저장
- 이어서(C)·광고그룹(G)·프로그램(AIR)·광고(SBC)는 공익재난 대상에서 제외
- **공익/재난 월별·흘림자막 송출내역**은 파일이 아닌 **apst.db + ddr1.db**에서 집계 → 수동송출(DDR1) 공익/재난 소재도 합산
- 방송운행표에서는 '방송 종료 안내' 프로그램의 경우 소재종류와 무관하게 공익/재난 포함 시 공익재난으로 표기

### R-2. 공익/재난 분류 키워드 (환경설정)
- 소재명에 특정 문자열이 있으면 공익/재난에 포함(각각 별도), 특정 문자열이 있으면 제외
- 설정키: `gongik_include_keywords`, `jaenan_include_keywords`, `gongik_jaenan_exclude_keywords` (콤마·줄바꿈 구분)
- **근무자 모드**에서도 이 키워드 3개 항목만 별도 화면(`/worker-keywords`)에서 수정·저장 가능 (관리자 로그인 불필요, 다른 설정은 노출·변경 안 됨)

### R-3. FTP 파일 가져오기 (폴더 실시간 감시 대체)
- FTP 서버(host/port/user/password)에서 날짜별 apst/ddr1_log/cml 파일을 내려받아 적재
- 매일 지정 시각(`ftp_fetch_time`)에 전날 파일 자동 수집, 실패 시 2분 간격 최대 5회 재시도
- 파일적재 메뉴에서 날짜 지정 수동 수집, 달력에서 누락일(붉은 0) 클릭 재수집

### R-4. 접근 제한 (허용 IP 대역)
- 환경설정에서 허용 IP 대역/단일 IP 지정(`allowed_ip_ranges`), `0.0.0.0`=전체 허용, localhost 항상 허용, 저장 즉시 반영

### R-5. 방송운행표 표시·양식
- 소재종류 라벨: P→프로그램, SBC→광고, IDC/F/E→기타, PRM+'시보'→시보
- 소재제목에 독립 `ID` 토큰이 있으면 ID로 표기(방송개시/종료 방송국 ID)
- 화면 표를 저장본과 동일 스타일(프로그램=하늘색, 광고/광고그룹=연두색, 프로그램/방송종료 경계·헤더 굵은 선), 단 '방송순서 안내'는 음영 제외
- PDF 회색 교차 음영 제거, 검색 결과 전체를 한 페이지에 표시

### R-6. 엑셀(xlsx) 저장
- 대시보드·달력·상세조회·소재목록 검색 결과 저장을 CSV → **xlsx**로 통일(서버 openpyxl 생성)

### R-7. 소재별 월 리포트 / 메뉴 정리
- 광고주 관리 메뉴 삭제, 월 리포트는 송출매체·비고 기본값 사용 + 저장 시 '송출 내용' 입력
- `report_dir`(미사용) 설정 제거, 환경설정 섹션 순서·구분선 정리

### R-8. 회사명·약칭 환경설정 (지역별 방송사 대응)
- 환경설정에 **회사명**(예: 광주문화방송)·**약칭**(예: 광주MBC) 항목 (기본값 광주문화방송/광주MBC)
- **약칭** → 좌측 로고, 브라우저 탭 제목, 월 리포트 문서 제목
- **회사명** → 월 리포트 '회사명' 칸, 하단 「○○(주)」
- 부산MBC·목포MBC·MBC경남 등 자유롭게 표기 가능

### R-9. 운영 편의
- 로그 기록 화면에 **로그 초기화**(관리자 전용) 버튼
- 페이지당 표시 줄 수 콤보박스가 실제로 동작(대시보드·상세조회·공익재난 월별)
- 폴더 감시 ON 상태는 **재시작/재부팅 후에도 유지**(자동 재개)
- 관리자 최초 기본 비밀번호: `admin2450` (신규 설치 시)

### R-10. 사내 서버(Ubuntu) 배포
- Ubuntu 초기 셋팅·앱 설치·데이터 전송 스크립트 제공(`init_ubuntu.sh`, `setup_ubuntu.sh`, `wsl_to_ubuntu_rsync.sh`)
- 하나의 서버(포트 8000)가 화면+API 제공, systemd 부팅 자동시작

### R-11. 소재별 월 리포트 문서 레이아웃 (PDF·Word)
- **위/아래 여백 동일(12mm)** + 표 행 높이를 자동 계산해 **한 페이지를 균형 있게 채움**(내용이 많으면 줄간격 자동 축소, 넘침 방지)
- Word는 정보표·데이터표·푸터를 모두 **가운데 정렬**해 좌우 여백 동일, **푸터(확인 문구 + 회사명)를 한 줄**로 배치
- **시간 표기 통일**: 새벽 0~4시를 `24:xx`(+24 변환)가 아니라 **실제 시각 `00:xx`** 로 표기(화면·파일 동일)

### R-12. 수동송출(DDR1) 소재명 보완
- CML에 없는 clip_id의 소재명을 자동송출(APST) 기록에서 조회해 보완
- **과거뿐 아니라 미래 APST 기록까지** 조회(신소재가 수동으로 먼저 나가고 자동편성은 이후인 경우 대응)
- 이미 적재된 과거 N-코드 행 일괄 보완용 스크립트(`backfill_manual_names.py`) 제공

### R-13. APST 파일명 접미사 설정 (`apst_suffix`)
- 파일명 `<YYYYMMDD><접미사>.apst`(예: `20260720AAA.apst`)의 접미사를 환경설정에서 지정(기본 `AAA`)
- **대소문자 무시** 매칭: `aaa` 입력 시 `...AAA.apst`/`...aaa.apst`, `a` 입력 시 `...A.apst`/`...a.apst` 를 읽음. 확장자 `.apst`/`.APST` 모두 허용

### R-14. 달력 붉은색(누락) 판정 기준
- 붉은색 = **APST 또는 DDR1 로그 파일이 없는 날** (CML 부재는 붉은색 사유 아님 — 소재명은 APST로 폴백)
- 추가로, **소재명을 끝내 못 찾은 수동송출(N-코드)이 남은 날**도 붉은색으로 표시
- 폴더 전체 스캔 시에도 처리한 날짜의 붉은/파랑 상태가 갱신됨
- CML 폴더도 실시간 감시 대상에 포함(늦게 도착한 CML도 상태 재판정 트리거)

### R-15. 파일 적재 행 단위 중복 방지
- 파일 적재(APST) 시 **파일명이 달라도 같은 (방송일·방송시간·clip_id) 행은 자동으로 건너뛰고**, 나머지 내용만 적재한다. (에러로 멈추지 않음)
  - 예: 오전만 들어간 날짜에 오전+오후 합본 파일을 이름만 바꿔 올리면, 오전분은 건너뛰고 오후분만 추가됨
- 같은 **파일명** 재업로드는 기존대로 차단(409)
- 손상/병합 오류로 JSON이 깨진 파일 업로드 시, "처리 실패" 대신 **구체적 사유 메시지**를 표시(파싱 오류 원인)
